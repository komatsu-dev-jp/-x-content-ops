## 推奨投稿（本命）

### Post 1｜7/2(木) — 問題提起型（Lane A）

```
勝った日の記憶って、なぜか鮮明に残る。

でも、その記憶でまた同じ台に座ると、だいたい根拠が後付けになる。

勝ち負けじゃなく「判断」を残して見直したくて、PachiTrackerを作っています。

#パチンコ #個人開発
```

- reason: 記憶バイアスという普遍痛点を1行目で提示。「勝ち負けではなく判断」で思想に直結し、勝ち保証から距離を取れている。
- risk: `none`
- image suggestion: なし（本文完結型）
- recommended time slot: **朝 08:00〜09:30**（分析・思想枠）

## 代替案

### Post 2｜7/3(金) — あるある型（Lane B）

```
これ、自分もやってた。

・この店は夕方から出る気がする
・あの島は据え置きっぽい
・前にここで勝った

全部「覚えてるつもり」。記録してみると、思ったほど当たってない。
```

- score: 85/100（validate: pass）

- risk: `none`
- image suggestion: なし
- recommended time slot: **昼 12:00〜13:00**（軽いあるある枠）

### Post 3｜7/4(土) — 失敗告白型（Lane C）

```
便利だろうと機能を増やしたら、自分が一番使わなくなった。

結局、実戦中に開くのは記録と判断ログだけだった。

PachiTrackerは多機能より、迷わず続けられることを優先して削っています。
```

- score: 85/100（validate: pass）
- reason: 「多機能で逆に使いづらい」失敗を開発者視点で告白。build-in-publicの信頼形成＋設計思想を提示。
- risk: `none`
- image suggestion: 任意で記録/判断ログ画面（権利・個人情報・店舗名なしを確認）
- recommended time slot: **朝 08:00〜09:30**（開発・思想枠）

### Post 4｜7/6(月) — 検証メモ型（Lane E）

```
「この台、回ってる気がする」

そう感じた時ほど、あとで記録を見るとボーダー前後でブレてることが多い。

印象じゃなく、自分の残した数字で続行か撤退かを見直したい。
```

- score: 85/100（validate: pass）
- reason: 「回ってる気がする」という現場の口癖を起点に印象と数字のズレを提示。内部閾値は出さず、振り返り思想に接続。
- risk: `none`（※ボーダーは一般用語、内部ロジックには触れていない）
- image suggestion: なし、または収支/回転率カード
- recommended time slot: **夜 21:00〜22:30**（実戦者・信頼枠）

### Post 5｜7/7(火) — UIスクショ型（Lane D）

```
続行か、撤退か。

その場の判断を、あとで一覧で見直せるようにした。

「なぜ続けたか」が残ると、次の判断が変わる。PachiTrackerの判断ログ画面です。

#個人開発
```

- score: 85/100（validate: pass）
- reason: 続行/撤退という最大の迷いどころを実画面で提示。プロフィール遷移期待が高い。
- risk: `none`
- image suggestion: **判断ログ画面のスクショ必須**（店舗名・個人情報・他社UI流用・未実装機能の写り込みを確認）
- recommended time slot: **夜 21:00〜22:30**（UI相談・実戦者枠）
