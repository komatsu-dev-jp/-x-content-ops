# 投稿フォーマットの学習

正典は `prompts/post-patterns.md`（型）・`prompts/scoring-rubric.md`（採点）・`prompts/banned-patterns.md`（禁止）。
ここには「繰り返しがちな逸脱」だけを足す。

- 1行目に痛点を置く。機能説明から始めない（Bad:「期待値分析アプリです」/ Good:「回ってる気がする、を後で数字で見直したい」）。
- ハッシュタグ0〜2個。リンク主役にしない。
- 「勝てる/稼げる/必勝/攻略」等は使わない（`banned-patterns.md`）。
- 店舗挙動を事実化しない（オカルト回避）。「出る」ではなく「勝った記憶がある」。
- 未実装機能を「できる」と断定しない。「〜する予定」と書く。

**Why:** Phase 0 は同一読者に何度も見られるため、既視感と売り込み臭が一番効く毒になる。
**How:** 生成後、各Skillのセルフチェックに加えてこの5点を確認する。
