# 検証・事実確認の学習

## それっぽい穴埋めをしない

- 失敗の根: 確証がないのに、文脈から「たぶんこう」で固有名詞・用語・数字を埋めてしまう。
- **Why:** 一見もっともらしいが事実が違うと、信頼を一度で失う（特にパチンコ実機・データに詳しい読者層）。
- **How:**
  - 確証がないものは「未確認」と明示するか、本人/一次情報に確認する。
  - パチンコ用語・機種挙動・規制・価格・実装状況は思い込みで断定しない。
  - X投稿のURL・数字は実在/実装を確認してから出す。

## 統計を語るときの最低ルール

- 最低母数（インプレッション500未満は保留）、経験ベイズ縮小、交絡（型×時間帯）を踏まえる。
- 少ない母数で「この型が勝ち」と断定しない（`x-weekly-review` の統計ルール）。

## 完成報告の条件

- 検証ステップ（文字数・型・禁止表現・事実確認）を通す前に「完成しました」と言わない。
