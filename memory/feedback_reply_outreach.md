# リプ周り・返信の学習

## 1. パチンコ用語・当たりの良し悪しを事実確認してから返信する

- 失敗例: 「チャージあたり」を“良い当たり”と誤読し、嘆きツイートに祝福調で返信案を作った。
  実際は「チャージ＝出玉のしょぼい最悪の当たり」で、相手は嘆いていた。
- **Why:** 用語の良し悪しを取り違えると、共感どころか相手を逆撫でする。EV/データ系の相手には特に「分かってない人」と見られ信頼を失う。
- **How:** 機種名・予告・当たりの種類・店舗挙動の意味に確証がなければ、断定せず本人に確認する。曖昧なまま「良かったですね/最悪ですね」のトーンを決めない。

## 2. 嘆き・自虐ツイートに「おめでとう/羨ましい」は厳禁

- 投稿の感情（自慢か嘆きか）を先に判定してからトーンを決める。`...` や `^_^` 等のニュアンスも読む。

## 3. 閉じた会話は無理に開けない

- `^_^`＋過去形（「移動しました」）の自己完結報告は実質クローズ。質問リプを投げると不自然。
- **How:** クローズ気味の投稿は「いいね＋フォローして見送り」。相手の“迷い・愚痴・問い”が出た開いた投稿で初めて絡む。リプ周りは数より「開いてる投稿を見抜く」のが効率の肝。

## 4. コールド返信に宣伝・リンク・アプリ名を入れない

- 相手が興味を示してから初めてPachiTrackerを出す（`reply-outreach-playbook.md`）。
