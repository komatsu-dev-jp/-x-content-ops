# Post Patterns

## 1. 問題提起型

目的：共感、返信、プロフィール遷移。

構造：

```text
パチンコで負ける原因は、
〇〇だけじゃない。

本当に怖いのは、
△△してしまうこと。

この判断を見直すために、
PachiTrackerを作っています。
```

## 2. あるある型

目的：軽い拡散、読者の自分事化。

構造：

```text
これ、やりがち。

・昨日出ていた台に座る
・前半回った記憶で続ける
・投資したからヤメられない

全部、判断が歪んでいるサイン。
```

## 3. 失敗告白型

目的：人間味、信頼、開発者感。

構造：

```text
自分で記録アプリを作っているのに、
記録をサボって判断ミスしました。

だから、入力をもっと軽くする必要がある。

便利な機能より、続く設計を優先します。
```

## 4. UIスクショ型

目的：プロフィール遷移、プロダクト理解。

構造：

```text
入力が面倒だと、
正しい判断より「楽な判断」を選ぶ。

だからPachiTrackerは、
実戦中に迷わない入力導線を優先しています。
```

## 5. 検証メモ型

目的：信頼、保存、リポスト。

構造：

```text
感覚で「回ってる」と思った台ほど、
あとで記録を見るとブレていることが多い。

記憶ではなく、
数字で振り返る癖を作りたい。
```

## 6. β募集型

目的：見込みユーザー獲得。

構造：

```text
パチンコの台選び・続行・撤退を、
感覚ではなく記録で見直したい人向けに
PachiTrackerを作っています。

β版を触ってくれる人を少人数で募集予定です。
```

## Hook Bank

- 台選びを勘でやると、あとで毎回ストーリーが変わる。
- 勝った日の記憶だけ残るの、かなり危ない。
- 入力が面倒だと、正しい判断より「楽な判断」を選ぶ。
- 自分用に作った記録アプリ、いちばん離脱したのは自分でした。
- β版、便利機能を増やしたら逆に使いづらくなりました。
- パチンコの反省、感情で振り返ると毎回ズレる。
- 「残した数字」と「頭の中の印象」が合ってない瞬間がいちばん怖い。
