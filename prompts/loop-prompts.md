# /loop プロンプト集 — PachiTracker X運用版

Claude Code の `/loop` に貼って使う実戦テンプレ。`<<...>>` を環境に合わせて書き換える。
（出典: ループエンジニアリングのテンプレを PachiTracker 運用に移植）

良いループの条件4つ: **何を（ゴール）／どのSkillで／どう検証して／いつ終わるか（終了条件）** が全部入っていること。

> **公開境界（厳守）**: どのループも「生成・分析・下書き保存まで」。Xへの投稿・返信送信・コミット/プッシュは人間が承認して行う（`CLAUDE.md`）。

---

## 1. 週次投稿バッチ（来週分の候補をまとめて作る）

```
/loop
x-growth-company を使って、来週のPachiTracker用X投稿候補を5本作って。

- 切り口は5角度（記憶のズレ/入力の面倒さ/続行理由/店・台選び/UI思想）から分散させる
- 各案に score / risk / 推奨タイムスロット / CSVログ用メモ を付ける
- banned-patterns と scoring-rubric のセルフチェックを必ず通す
- 最大5本できたら output/ に保存してレポートを出して停止
公開はしない（人間が選んで手動投稿）。
```

## 2. ネタ収集（backlogを増やす）

```
/loop
x-idea-harvest を使って、idea_backlog.csv に投稿ネタを10件追加して。

- 既存ネタと重複しないこと。角度が偏らないこと
- 1件ずつ「痛点 / 角度 / 想定型」を付ける
- 10件追加し終わったら停止
```

## 3. リプ周り準備（開いてる投稿を見抜いて返信案を作る）

```
/loop
渡す投稿（リンク or 本文）について x-reply-outreach で返信案を3つ作って。

- まず「会話が開いているか（迷い・問い・愚痴か）」を判定。閉じていれば返信非推奨と理由を述べる
- パチンコ用語・当たりの良し悪しは確証がなければ確認する（memory/feedback_reply_outreach.md）
- リンク・宣伝は入れない
返信案を出したら停止。送信は人間が行う。
```

## 4. 週次レビュー → 学習（改善ループ）

```
/loop
npm run weekly の結果（data/post_log.csv）を x-weekly-review で解釈して。

- 最低母数・経験ベイズ縮小・交絡を踏まえ、「勝ち仮説」を1つだけ出す
- 同じ改善指摘が2回出ていれば memory/feedback_*.md に追記し MEMORY.md に索引を足す
- 来週のA/B変数（1つだけ）を data/ab_test_plan.csv 方針に沿って提案
分析→記録→次週仮説まで出したら停止。
```

## 5. ゴール駆動（/goal：達成まで自走）

```
/goal
来週分のX投稿カレンダーを「予約できる状態」まで仕上げる。

達成条件:
- 候補5本が score 90以上 / risk none
- 各案に曜日・タイムスロット・画像有無が割り当て済み（ab_test_plan.csv 準拠）
- npm run schedule で予約用CSVが output/ に書き出されている
達成したら要約を出して終了。公開はしない。
```

---

## アレンジのコツ

- 動かないときはまず「ゴールが曖昧でないか」を疑う（×「投稿がんばって」→ ◯「候補5本をscore90で下書きまで」）。
- 検証項目を1つ足すごとに、放置できる時間が伸びる。
- 公開・課金・削除・プッシュは必ず「人間に確認してから」と書く。
