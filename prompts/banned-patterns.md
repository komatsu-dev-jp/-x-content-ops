# Banned Patterns

## 禁止表現

- 絶対勝てる
- 勝率爆上げ
- 誰でも稼げる
- パチンコで生活できる
- 必勝
- 攻略法
- 爆益
- 最強
- 革命
- 人生変わる
- 今すぐ登録
- これを使わない人は損
- 勝ち組になれる
- 期待値だけで勝てる

## 避けたい投稿パターン

### 1. 勝ち保証型

Bad:

```text
PachiTrackerを使えば勝率が上がります。
```

Good:

```text
PachiTrackerは、判断を後から見直すための記録アプリです。
```

### 2. 機能説明だけ型

Bad:

```text
回転率分析、収支管理、期待値計算を実装しました。
```

Good:

```text
「回ってる気がする」を、あとで数字で見直せるようにしました。
```

### 3. 煽り型

Bad:

```text
まだ感覚で打ってる人は危険です。
```

Good:

```text
感覚で打つと、あとで判断理由が変わることがあります。
```

### 4. 内部ロジック漏洩型

Bad:

```text
ベイズ推定でこの閾値を超えたら撤退判定します。
```

Good:

```text
判断に迷う場面を、後から見直せるようにしています。
```

## 注意表現（rejectではないが必ず言い換える）

採点では落ちにくいが、信頼や正確性を損なう表現。`x-risk-check` で caution 扱い。

### A. 未実装断定

未提供の機能を「できる」と断定しない。Phase 0 は特に注意。

Bad: `あとで振り返れる部分から試せます。`
Good: `あとで振り返れる部分から、試してもらう予定です。`

### B. オカルト・迷信（店側挙動の事実化）

店の出玉挙動を事実のように書かない。記憶・印象として書く。

Bad: `この店は夕方から出る` / `あの島は据え置きっぽい`
Good: `前に勝った記憶で、また同じ店に行く` / `この島は相性がいい気がする`

### C. 専門語の壁

初心者に伝わらない専門語は日常語へ。

Bad: `ボーダー前後でブレてる`
Good: `続ける根拠としては弱かった`

### D. UI動作の誇張

実装と一致しない操作を断定しない。

Bad: `その場で一押し`（実装がワンタップでない場合）
Good: `その場でサッと記録`
