# x-reply-outreach

## Purpose

他人の投稿へのコールド返信で、相手とそのフォロワーをPachiTrackerに呼び込む。
「返信が返ってくる」ことを最優先に、短く価値のある返信案を作る。**自動投稿しない。**

## Read first

- `prompts/reply-outreach-playbook.md`（最重要：返信が返る原則・型・禁止事項）
- `prompts/pachitracker-positioning.md`（思想を“にじませる”程度に使う）
- `prompts/banned-patterns.md`（禁止表現）

## Inputs

ユーザーは通常 **ターゲット投稿のリンクだけ** を渡す。次の順で本文を得る:

1. `python3 scripts/fetch_tweet.py <URL>` を試す（ミラー経由の best-effort）。
   取得できたら本文・投稿者・画像有無を使う。
2. 取得不可（X本体は未ログインだと403、ネットワーク制限もある）なら、
   **「本文をコピペしてください」と1回だけ依頼する**。憶測で本文を作らない。

任意で受け取ると精度が上がる情報: 投稿者の属性（規模/界隈）、こちらの狙い（認知/フォロー/会話）。

## Workflow

1. ターゲット本文を確定（fetch or 貼付）。空なら本文を依頼して停止。
2. **関連性ゲート**: パチンコの判断・記録・データ文脈に重なるか判定。
   重ならない投稿なら「関連が薄いので返信非推奨」と理由を述べて止める。
3. 投稿の意図を1行で要約（主張/問題提起/感想/質問のどれか）。
4. Playbook の型から、この投稿に効く型を2〜3種選ぶ。
5. 各型で返信案を作る（**各1〜2文、リンクなし、宣伝なし、相手の語を1つ拾う**）。
6. `x-risk-check` 観点で自己チェック（断定・煽り・晒し・規約・宣伝臭を除去）。
7. 推奨1案を選び、なぜ返信が返りやすいかを一言添える。

## Output Format

```markdown
# リプ案：<投稿者 or URL>

## 相手投稿の要約
- 一行要約（主張/問題提起/感想/質問）
- 関連性: 高 / 中 / 低（低なら返信非推奨の理由）

## 推奨リプ（型: ○○）
本文（1〜2文）
- 狙い: なぜ返信が返りやすいか（1行）

## 代替案
### 案2（型: ○○）
本文
- 狙い:
### 案3（型: ○○）
本文
- 狙い:

## やらない/注意
- リンク・宣伝は入れていない（コールド返信のため）
- 必要なら本文の事実確認を
```

## 投稿後（任意・進捗カウント）

人間が選んだ返信を手で送信したら、その1件を本日のタスクに数える:

```
npm run today -- --done-reply target_url=<相手投稿URL> archetype=<使った型> [got_reply=0 profile_visit=0 follow=0 note=...]
```

→ `本日のタスク リプ周り 2/5 あと3件` のように進捗が出る（目標は `data/daily_goals.json`）。
進捗だけ見たいときは `npm run today`。**送信は人間。スクリプトは数えるだけで投稿しない。**

## Hard Constraints

- 自動返信・自動巡回をしない（候補のみ。人間が選んで手動送信）
- コールド返信にリンク・アプリ名・β募集を入れない（相手が興味を示すまで）
- 「勝てる/稼げる」断定、当確煽り、相手や第三者の晒し、内部情報を出さない
- 関連性の薄い投稿・炎上スレには返信しない
- 1日の返信は質重視で3〜7件まで（連投巡回に見せない）
- 本文が取れないときに内容を捏造しない
