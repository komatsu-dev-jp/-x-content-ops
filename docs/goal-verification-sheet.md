# ワークフロー別 ゴール・検証シート

ループ設計の核は「ゴール」と「検証」のセット。PachiTracker X運用の各ワークフローの早見表。
（出典: ループエンジニアリングの goal-verification-sheet を本運用に移植）

公開方式はすべて **「生成・分析・下書きまで自動／公開と送信は人間」**（`CLAUDE.md` の公開境界）。

| ワークフロー | ゴール（何を作って終わりか） | 検証（AIが自分で確かめる） | 公開方式 |
|---|---|---|---|
| 投稿候補生成（x-growth-company / x-draft-post） | 候補3〜5本＋score＋risk＋推奨スロット | 文字数・1行目フック・型・banned-patterns・positioning・「AIマーケ臭くないか」 | 下書き保存 → **人間が手動投稿/X Pro予約** |
| ネタ収集（x-idea-harvest） | idea_backlog.csv に重複なしでN件追加 | 既存との重複・角度の偏り | 自動追記（CSV） |
| 採点（x-score-post） | 各候補を100点で採点 | scoring-rubric準拠・90↑投稿可/80↑要改善 | レポートのみ |
| リスク確認（x-risk-check） | safe / caution / reject 判定 | 規約・景表法・煽り・未実装断定・内部漏洩 | レポートのみ |
| リプ周り（x-reply-outreach） | 返信案3つ＋推奨1つ | 会話が開いているか・用語の良し悪し確認・リンク/宣伝なし | 候補提示 → **送信は人間** |
| 週次レビュー（x-weekly-review / npm run weekly） | 勝ち仮説1つ＋次週A/B変数1つ | 最低母数・経験ベイズ縮小・交絡 | レポート＋memory追記 |
| 予約準備（npm run schedule） | 予約用ワークシートCSV | 曜日・スロット・画像有無の割当（ab_test_plan準拠） | CSV書き出し → **予約は人間** |

## 使い方

1. まず1ワークフローだけ選ぶ（おすすめは週次レビュー：レポートで完結し事故が起きない）。
2. 表の「ゴール」と「検証」を `prompts/loop-prompts.md` のテンプレに書き写す。
3. 1〜2週回して、修正指示した内容を `memory/feedback_*.md` に蓄積する。
4. 安定したら次のワークフローを足す。

## ゴール設計の良し悪し

| | 例 | 評価 |
|---|---|---|
| ✕ ダメ | 「Xがんばって」 | 曖昧・終わりがない・検証不能 → 暴走する |
| ◯ 良い | 「候補5本（score90以上・risk none・スロット割当済み）を下書き保存まで」 | 具体・検証可能・終了条件あり |
